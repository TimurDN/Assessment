# Bug Reports — Booking Flow

> This file will contain ≥ 3 documented bugs surfaced during manual and automated testing.
>
> **Required (per assessment):**
> - Minimum 3 bugs
> - Each bug: severity, priority, environment, steps to reproduce, actual vs expected, evidence (screenshot/video/log)
> - At least 1 bug must involve branching or state-transition logic
>
> _Populated during the testing phase._

## Template

```markdown
### BUG-XXX — <short title>

- **Severity:** Critical / High / Medium / Low
- **Priority:** P1 / P2 / P3
- **Environment:** Browser + version, OS, viewport, URL/commit
- **Category:** Functional / UI / State / API / Accessibility / Performance
- **Steps to reproduce:**
  1. ...
  2. ...
- **Expected result:** ...
- **Actual result:** ...
- **Evidence:** `docs/bugs/BUG-XXX.png` (or video/log path)
- **Notes:** Workaround, suspected root cause, related tickets
```
